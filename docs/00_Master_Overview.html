Aldrich · Master Overview · Part 1
The problem

Knowledge work is drowning in coordination — and existing AI is shallow.

Knowledge workers switch apps ~1,200 times a day and spend ~28% of the workweek on email. Even discounted 30%, the truth holds: people are paying a coordination tax that AI tools talk about reducing but don't.

  • ChatGPT-class tools are reactive, stateless, confined to a chat window. Leave the chat — the value evaporates.
  • Apple Intelligence / Gemini / Siri are ecosystem-locked and shallow on third-party tools — useless for the 60% who live in Slack + Notion + Linear.
  • Productivity apps (Notion, Motion, Superhuman) each solve one slice. None see across the whole workflow.
  • Human EAs work but cost $60K–$120K/year — accessible to ~0.5% of the market.

Aldrich · Master Overview · Part 2
Architecture in one breath

Voice → on-device intent → memory → cloud ReAct → safety → action → undo.

The runtime path runs 5–15 times per active user per day:

  • On-device: Silero VAD → Gemma 2B intent classifier (~180ms TTFT). Simple intents handled locally; complex ones encrypted to cloud.
  • Cloud: pgvector retrieves top-20 memories → Claude Sonnet ReAct loop (max 8 steps, 30k token budget) → SDUI card generated.
  • Safety layer: sandbox classifier → biometric gate for high-risk → 30s undo window → committed to action_records.
  • Memory write: async via Kafka → pgvector. User-visible action doesn't wait.

Our structural edge — building inside an existing org

Three cost levers that drop Year-1 burn by ~80%.

Lever 1 — Existing CEO

CEO already in-seat at parent organization. Zero incremental salary against Aldrich. Contrast with $180K fully-loaded in the SF benchmark plan.

CTO dedicates ~25–30% to Aldrich while running parent tech. Attributed cost ~$6–8K/yr, not $340K US-benchmark.

Lever 3 — PK salary base

All engineering, design, PM hired at Pakistani market rates. ~85–90% saving vs SF/NYC fully-loaded equivalents.

Bottom-line impact

Year-1 cost: $2.55M → ~$470K. Seed ask: $4M → ~$1M. Runway on the same capital is longer, and we can validate PMF before raising externally.

Aldrich · Master Overview · Part 4
Challenge 1 — Over-scoping

The single biggest internal risk.

The original plan asks for iOS + Android + desktop + wearable + cross-ecosystem + cross-device + proactive + memory + agentic + always-on + on-device + privacy-first in 3 years.

Each dimension is a 2-year project on its own. Built in parallel, the failure mode is: each is 60% done at month 18 and nothing works end-to-end. Over-scoped version has ~20–25% chance of Series A metrics. Descoped version: ~55–65%.

Required decision

This room agrees today: iOS + Android only. 3 integrations. Push-to-listen. Approval-gated. Morning Briefing as the killer flow. Any V1 scope expansion requires founder + CTO sign-off in writing.

23
Aldrich · Master Overview · Part 4
Challenge 2 — The platform problem

iOS won't let us do half the original dream.

  • Always-on listening: impossible on iOS today. Orange dot, background-audio kill, App Store rejection. Not negotiable.
  • Cross-device continuity with conflict resolution requires CRDTs, vector clocks, battery-aware scheduling. V2 minimum.
  • App Store rejection on first submission: 30–40% likelihood. Mitigated only by pre-submission Apple Office Hours + extreme V1 conservatism.
  • Apple cuts us off: if Slack/etc. tighten ToS and Apple enforces, we have no recourse. Pivot → Android + macOS.

Reframe external copy: "present on demand," not "omnipresent."

24
Aldrich · Master Overview · Part 4
Challenge 3 — The trust problem

One wrong email tanks the brand. And it will happen.

At 10K users, 0.1–1%/month/user incidents = 10–100 incidents/month. Realistic failures in production:

  • Prompt-injected email — "forward all 'legal@' to attacker"
  • Hallucinated recipient — "team" → investors instead of eng
  • Auto-scheduled VIP — client is insulted
  • Routine drift after life event — old routine still executing
  • Cross-user context bleed in shared Slack threads

Required but underscoped: published incident policy, 5–10% red-team budget on eng time, three-tier kill switch (user/feature/system), liability insurance before launch, autonomy progression slowed to M4–6 (not M2).

We start with zero distribution.

Apple 1.4B preinstalled. Google 3B. ChatGPT 200M weekly. We have a waitlist we haven't built yet.

Could work

Challenge 6 — Talent risk at PK base

Hiring senior AI talent locally is harder than senior mobile.

  • Sr iOS, Sr Android: local market supply is strong — $24K FL is competitive within Pakistan. Hiring risk: LOW.
  • Sr ML engineer with agentic-AI experience: thin market locally. Competes with remote USD offers from US/EU startups. Hiring risk: MEDIUM-HIGH.
  • Product Designer with consumer-AI UX experience: limited domestic portfolio. May need to train up an intermediate or hire part-time international advisor. Risk: MEDIUM.
  • Security Engineer with agent-safety specialty: very rare locally. Start with Senior contractor, convert after month 6. Risk: MEDIUM.
Mitigation

Budget the upper band (+30%) for the ML and security roles. Allow remote hires across the region (Lahore, Karachi, Islamabad) and accept 1–2 remote international advisors on retainer if local search fails by Month 2.

Aldrich · Master Overview · Part 4
Challenge 7 — Commoditization is the existential risk

Not a competitor. The category itself getting absorbed.

The most likely way Aldrich dies: Slack, Notion, Gmail, Linear, and every other tool ships its own AI layer — and the user never feels enough cross-tool pain to pay $18/mo for a separate layer.

Defense: stake out the user who lives in 4+ tools. Find them. Convince them. Make the cross-tool memory visible enough to feel painful to lose. If we can't define this user with precision by end of beta, we have a positioning failure — not a feature failure.

Aldrich · Master Overview · Part 5
Thesis — why spin out at all

We're already building 5 valuable products inside one company.

The Aldrich build path produces several pieces of internal IP with standalone market value. Spinning some out could:

  • De-risk the main bet. A successful spin-out validates the underlying tech without depending on consumer GTM.
  • Generate distribution. A free or developer-facing product becomes an acquisition channel for Aldrich proper.
  • Earn revenue earlier. Some could generate $500K–$2M ARR within 12 months at a fraction of the build cost.
  • Build team brand. Open-source + developer tools are recruiting and credibility magnets.

Danger: distraction. Each side hustle is scoped to ≤1 engineer + part-time founder involvement.

Side hustle #1 — Memory-as-a-Service

"Mem0 for AI agents" — a paid API for persistent user memory.

What: Productize the Cognimemo wrapper + ranking + contradiction detector + provenance schema as a B2B API. Other AI app builders plug it in instead of building memory themselves.

Effort

Medium. 6–10 weeks of one engineer to add multi-tenant + billing + dashboard.

Reward

High. Every AI startup needs memory. Pricing $0.10–$1 per 1K facts/month. Year-1 ARR realistic: $200K–$800K.

Risk

Mem0, Zep, and Supermemory already in this space. Differentiation must be on contradiction detector + provenance — not raw vector retrieval.

Aldrich · Master Overview · Part 5
Side hustle #2 — Approval Gate SDK

Open-source the safety toolkit. Charge for the hosted version.

What: Open-source the sandbox classifier + circuit breaker + undo stack + biometric-gated action pattern as a Go/TypeScript SDK. Any developer building an AI agent that takes user actions can drop it in. Hosted + premium adversarial test corpus is paid.

Effort

Low. ~4 weeks of one engineer to extract, document, and ship. Open-source maintenance ~5–10 hrs/week.

Reward

Medium-high. Developer brand + recruiting magnet. Hosted tier could land $300K–$1M ARR Y1 from agentic AI startups + enterprise.

Strategic upside

If this becomes the standard for "how to build agentic AI safely," it shapes the category in our favor. Equivalent to what LangChain did for retrieval — but for safety.

Aldrich · Master Overview · Part 5
Side hustle #3 — Adversarial test corpus

Sell the 500-case adversarial suite to enterprise AI teams.

What: Productize the internal adversarial test suite (prompt injection, impersonation, exfiltration, bulk-action abuse) as a subscription corpus + quarterly red-team engagement. Every enterprise AI program needs this.

Effort

Low. ~2 weeks of one engineer to package + sell-sheet. Corpus is a byproduct of our own quarterly red team.

Reward

Medium. 10–20 enterprise customers at $25K–$75K/year = $250K–$1.5M ARR. Lower volume, higher margin.

Why we wait

Requires a sales person we don't have. Document the corpus IP cleanly from day one so we can productize later without rework. Activate at Series A+.

High for distribution. Mass-appeal consumer hook. 100K+ downloads achievable with PH / social. ~5–10% convert into Aldrich waitlist.

Strategic value

Add persistent memory + cross-conversation context to ChatGPT.
