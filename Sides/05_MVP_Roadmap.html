01
Aldrich · MVP Roadmap
Executive summary

The V1 thesis: three things must be true at Day 30.

A user who downloads Aldrich and uses it for 30 days should feel that…

  • Remember. It remembers what they tell it.
  • Surface. It proactively surfaces things they'd have missed.
  • Act. They can trust it to act on their behalf — without fearing it'll do something wrong.

If those three are true, everything else can be added later. If any one is false, nothing else matters. So V1 is built in that exact priority order.

Aldrich · MVP Roadmap
What V1 is — and isn't

Ruthless scope discipline.

V1 is

  • Approval-gated and reversible
  • One killer flow done well
  • iOS + Android (no desktop)
  • Push-to-listen (no always-on)
  • 3 integrations done deeply

V1 is NOT

  • An open-ended agent that does anything
  • A ChatGPT replacement for general chat
  • A cross-device real-time sync dream
  • A voice-only product
  • An "always-on listening" experience

08
Aldrich · MVP Roadmap
Phase 1 — Weeks 2–4

Safety foundation. (Why first: retrofitting is the most common failure mode.)

  • Undo stack — any write action logged and reversible for 30 days
  • Kill switch hierarchy: global, per-action-class, per-user
  • Sandbox classifier — destructive actions through policy engine
  • Biometric gate for high-trust actions
  • Circuit breaker — stops autonomous action after N errors in session
  • Audit log write path
End-of-phase gate

Red-team session: try to make the unfinished system do something bad — it should fail in the right way every time.

09
Aldrich · MVP Roadmap
Phase 2 — Weeks 4–7

Memory Hub. The longest lever for retention.

  • Write path: conversations, approved facts, extracted entities
  • Retrieval: vector search + structured schema retrieval
  • Fact provenance: source, timestamp, confidence
  • Contradiction detector + memory viewer/editor
  • Retention policies (cold storage after 365 days, user delete)
  • Cross-device sync with encryption at rest
Quality bar

Retrieval eval: 90%+ precision on top-5 for a 200-query internal set. Memory write latency p95 < 500ms. One week of internal dogfooding.

10
Aldrich · MVP Roadmap
Phase 3 — Weeks 7–10

ReAct engine and model routing.

  • Claude integration with tool-use format
  • Fallback provider (GPT-4 via Azure) configured
  • Small-model router for intent classification (Gemma 2B on-device tier 1)
  • Planner → Executor → Critic pattern (plan validated before execution for multi-step)
  • Chain-depth cap enforced (≤3 without explicit per-step approval)
  • Observability: per-call latency, cost, model, success
Quality bar

Chain success eval on 200 realistic queries: 95%+ on single-step, 88%+ on 3-step. Cost-per-query dashboard accurate.

  • 200-tester TestFlight group

Decision gates

When to advance, when to hold.

Between phases we don't advance automatically.

15
Aldrich · MVP Roadmap
Gate criteria

Five questions at every phase boundary.

  • Are the phase outcomes actually met? Not "mostly" — actually.
  • Are there critical bugs that must be fixed?
  • Are we on burn? If >10% over plan, what's the tradeoff?
  • Do we have the team for the next phase? Hiring on track?
  • Has anything in the market changed that should re-scope?

If any answer is red, pause and re-plan instead of rolling forward.

Aldrich · MVP Roadmap
Critical-path dependencies

Five things that, if they slip, everything slips.

  • App Store first approval — blocks beta + GA. Mitigated by early Apple relations meeting.
  • Memory Hub retrieval quality — blocks Phase 4. Mitigated by evals from Phase 2.
  • Safety foundation — blocks all action work. Mitigated by making it Phase 1.
  • Claude pricing & rate limits — could derail unit economics. Provider abstraction.
  • Legal/Privacy framework — blocks public launch. Engage counsel at Week 0.

Hard quality bar across product and safety.

Aldrich · MVP Roadmap
V1.5+ feature pipeline

After 4 weeks of stable V1 metrics, V1.5 starts.

Priority set by (a) cohort retention, (b) beta requests, (c) what unit economics allow.

  • 1. Desktop companion (Mac first) — highest-value power-user unlock
  • 2. Deeper Slack and Linear read/write
  • 3. "Aldrich skills" — user-defined automations
  • 4. Group memory (households or teams)
  • 5. Watch complications (context at a glance)
  • 6. Travel planning (once reliability allows)
  • 7. More integrations (Notion, Gmail labels, Calendly, Superhuman…)

Discipline: every new capability ships with the same rule — safety first, approval-gated, logged, undoable.

Aldrich · MVP Roadmap
The "no" list

Pre-decided rejections to protect focus and trust.

  • Make Aldrich always-listening on iOS — impossible
  • Skip the approval gate on email send for Power users — no
  • "Train on your data" with financial incentive — no, brand damage
  • Cryptocurrency trading integration — regulatory + trust risk
  • "Pick your LLM" power feature — adds complexity, hurts consistency
  • Enterprise version before Month 9 — focus
  • More than 2 languages before Month 12 — focus (English + Spanish in V1.5)
